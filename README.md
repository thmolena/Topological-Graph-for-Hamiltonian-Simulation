## Summary

Simulating the time evolution `e^{-iHt}` of a many-body Hamiltonian is a founding
application of quantum computers, and the first-order Lie–Trotter product formula remains
its dominant primitive. For a Hamiltonian in Pauli form `H = Σ_j c_j P_j`, the per-step
cost of a first-order schedule is exactly one rotation per term, so the gate count `L·r`
is invariant under the term ordering `π`; the algorithmic error, in contrast, is governed
by the commutators of the non-commuting summands and therefore depends on `π`. The term
ordering is thus a degree of freedom that lowers Trotter error while holding the gate
budget fixed.

This work casts first-order Trotter ordering as a problem on a **commutator graph**, in
which terms are nodes and an edge joins two terms that anticommute. To leading order the
ordering-dependent part of the error reduces to the coefficient-weighted sum over the
anticommuting edges that fall adjacent in the schedule, so the continuous fidelity
objective collapses to a combinatorial surrogate: minimize the coefficient-weighted count
of adjacent anticommuting pairs. A greedy commutator-walk policy drives that count toward
its minimum, and a compact learned router over commutator-graph features recovers the same
ordering signal. The effect is measured against exact statevector references on 120
structured Hamiltonian instances, with every quantity reported as a mean and a 95%
confidence interval.

Every number in this README is generated by the included code and transcribed verbatim
from `code/results/summary.json`; each value traces to that artifact rather than to manual
entry.

## Principal contributions

1. **Formulation.** First-order Trotter ordering is cast as minimizing the
   coefficient-weighted count of anticommuting adjacencies on the commutator graph
   `G(H)`. A greedy walk policy drives this count toward its minimum while holding the gate
   budget `L·r` fixed.
2. **Measurement.** The ordering effect is quantified against exact statevector references
   on 120 structured Hamiltonian instances, every quantity carrying a mean and a 95%
   confidence interval. This separates the statistically decisive ordering-versus-random
   gain from the within-confidence-interval spread among the structured heuristics.
3. **Learned policy.** A compact, interpretable router — logistic regression over eight
   commutator-graph features, trained leave-one-instance-out — matches the best fixed
   strategy. The match indicates that the graph features alone carry the ordering signal
   under a listing-order–invariant policy.

## Method

```
 H = Σ c_j P_j ──► commutator graph ──► ordering policy ──► first-order Trotter schedule
     (Pauli terms)   (edge ⇔ anticommute)  (commutator/learned)   (Π_j e^{-i c_j P_j t/r})^r
                          │                                              │
                          ▼                                              ▼
                  greedy commuting groups                    exact statevector simulator
                  + coefficient-weighted             ◄──── fidelity vs e^{-iHt} (expm / Krylov)
                  anticommutation cost                       at matched gate proxy = |terms|·r
```

1. **Pauli algebra** ([`pauli.py`](code/src/topoham/pauli.py)) — Pauli strings on `n` qubits with an `O(n)`
   commutation parity test, and the exact single-Pauli rotation `cos θ·ψ − i sin θ·(Pψ)`
   (valid because `P² = I`). A test asserts the kernel matches `scipy.linalg.expm(−iθP)`
   to `1e-9`, so the fast statevector Trotter path — and the in-browser demo — rests on a
   verified primitive.
2. **Commutator graph** ([`commutator_graph.py`](code/src/topoham/commutator_graph.py)) — nodes are terms, edges are
   anticommuting pairs. The module exposes anticommutation density, greedy
   commuting-group colouring, and the features the learned policy reads. Each feature is an
   invariant of the term *set*, so the policy's decision remains independent of how the
   terms are listed.
3. **Orderings** ([`policies.py`](code/src/topoham/policies.py)) — `random`, `coefficient` (by `|c_j|`),
   `locality` (by qubit support), `commutator` (the proposed policy), and `learned` (a
   scikit-learn router over commutator-graph features). All are compared at a matched gate
   budget `num_terms × r`.
4. **Audited reward** — fidelity of the Trotter state against the exact `e^{-iHt}`
   reference (`scipy.linalg.expm`, cross-checked against Krylov `expm_multiply`).

## Headline results

Reported scale: 120 structured instances (transverse-field Ising, Heisenberg, and random
molecular-like families; `n ∈ {4,…,8}` qubits; eight seeded draws each), evolution time
`t = 2.0`, `r = 3` steps, gate proxy `L·r = 48`. Transcribed verbatim from
[`code/results/summary.json`](code/results/summary.json).

| ordering | fidelity | ±95% CI | infidelity | adj. anticomm. | gate proxy |
|---|---|---|---|---|---|
| random | 0.2054 | 0.0400 | 0.7946 | 3.08 | 48 |
| coefficient | 0.5501 | 0.0670 | 0.4499 | 2.86 | 48 |
| locality | 0.5004 | 0.0684 | 0.4996 | 6.47 | 48 |
| **commutator** | 0.5481 | 0.0677 | 0.4519 | **0.28** | 48 |
| learned | 0.5486 | 0.0670 | 0.4514 | 2.50 | 48 |

Random ordering reaches `0.205 ± 0.040`; every structured ordering more than doubles it,
the commutator ordering landing at `0.548 ± 0.068`. The two intervals are disjoint and the
`0.343` mean gain spans many confidence-interval widths — equivalently a `1.76×` infidelity
reduction at the identical gate proxy of 48. The structured orderings remain statistically
indistinguishable at this scale (means within `0.002`, well inside their `≈ ±0.067`
intervals); they are separated by mechanism rather than fidelity — the commutator policy
attains that fidelity with by far the fewest anticommuting adjacencies (0.28 vs 3.08).

The fidelity–gate-budget frontier (sweeping `r ∈ {1,2,3,4,6,8,12}`) and the per-family
decomposition are written to `code/results/summary.json` and rendered in
`code/figures/`. On Heisenberg every structured ordering is exact (`1.000 ± 0.000`) where
random reaches only `0.169 ± 0.086`; on the dense molecular-like family all five orderings
fall in `0.15–0.17` with overlapping intervals — a reported result indicating that the
ordering lever is exhausted on dense, generic graphs at small `r`.

## Installation

```bash
pip install topoham
```

Or install the development checkout in editable form, which also exposes the reproduction
pipeline and the test suite:

```bash
git clone https://github.com/thmolena/P3-Lie-Topological-Graph-Policies-for-Hamiltonian-Simulation
cd P3-Lie-Topological-Graph-Policies-for-Hamiltonian-Simulation/code
pip install -e ".[dev]"
```

The implementation is CPU-only and depends on numpy, scipy, networkx, scikit-learn,
matplotlib, and pyyaml. Optional accelerated backends are declared under the `gpu` extra.

## Reproduction

A single command regenerates every figure, table, and headline number from one seeded run.
Invoke it from the repository's `code/` directory:

```bash
cd code
topoham-reproduce --config configs/full.yaml
```

The run is fully determined by seeded draws (`seed = 0`), so the reported numbers are
recovered exactly; only the provenance fields (timestamp, runtime, peak memory) vary
between runs. The equivalent Makefile path from a clean checkout is:

```bash
cd code
make test            # 24 tests incl. Pauli-rotation ↔ expm cross-check to 1e-9
make full-run        # reported scale: 120 instances → results/summary.json
make tables figures  # results/main_results.tex, figures/*.pdf
make audit           # readiness gate: traceable numbers, vetted claims
```

`make demo` substitutes the smoke configuration for a sub-minute check.

## Scope

The study is first-order (Lie–Trotter) only; the effect of ordering under higher-order
Suzuki formulas remains open. Systems are small (`n ≤ 8`) because an exact statevector
reference is formed; the at-scale effect, which would require tensor-network or Krylov
references, is left for future work. Families are structured proxies rather than
production-mapped molecular Hamiltonians, and cost is a proxy `L·r` that abstracts rotation
synthesis and device connectivity. Among the structured orderings the fidelity differences
fall within the confidence intervals at this scale; the ordering-versus-random effect and
the adjacency-count reduction are the statistically decisive findings. All results are
classical CPU statevector simulations, reported without hardware validation.
