## TL;DR

The cost of digital Hamiltonian simulation is dominated by **Trotter error at a fixed
gate budget**. In first-order Trotterization, the error between two terms is governed by
their **commutator** — it vanishes whenever the terms commute. The *order* in which the
per-term rotations are applied therefore changes the error **at zero extra gate cost**.
We treat the Hamiltonian's terms as nodes of a **commutator graph** (an edge ⇔ two terms
*anticommute*) and order them with a graph policy that minimizes the weighted count of
anticommuting adjacencies in the schedule. On transverse-field Ising, Heisenberg and
random molecular-like Hamiltonians, the **commutator ordering reaches higher fidelity than
random, coefficient- and locality-based orderings at a matched gate budget**, and a small
**learned router** over the commutator-graph features tracks the best fixed strategy.

Every number in this README is generated by the included code (`make demo`); nothing is hand-edited.

## Method

```
 H = Σ c_j P_j ──► commutator graph ──► ordering policy ──► first-order Trotter schedule
     (Pauli terms)   (edge ⇔ anticommute)  (commutator/learned)   (Π_j e^{-i c_j P_j t/r})^r
                          │                                              │
                          ▼                                              ▼
                  greedy commuting groups                    exact statevector simulator
                  + coefficient-weighted             ◄──── fidelity vs e^{-iHt} (expm / Krylov)
                  anticommutation cost                       at matched gate proxy = |terms|·r
```

1. **Pauli algebra** ([`pauli.py`](code/src/topoham/pauli.py)) — Pauli strings on `n` qubits with an O(n)
   commutation test, and the exact single-Pauli rotation `cos θ·ψ − i sin θ·(Pψ)` (valid
   because `P² = I`). A test asserts it matches `scipy.linalg.expm(−iθP)` to `1e-9`, so the
   fast statevector Trotter path — and the in-browser demo — is provably correct.
2. **Commutator graph** ([`commutator_graph.py`](code/src/topoham/commutator_graph.py)) — nodes are terms, edges are
   anticommuting pairs; exposes anticommutation density, greedy commuting-group colouring
   and the features the learned policy reads. Every feature is an invariant of the term
   *set*, so the policy's decision is independent of how the terms are listed.
3. **Orderings** ([`policies.py`](code/src/topoham/policies.py)) — `random`, `coefficient` (by `|c_j|`),
   `locality` (by qubit support), `commutator` (the contribution), and `learned` (a
   scikit-learn router over commutator-graph features). All compared at a **matched gate
   budget** `num_terms × r`.
4. **Audited reward** — fidelity of the Trotter state against the exact `e^{-iHt}`
   reference (`scipy.linalg.expm`, cross-checked against Krylov `expm_multiply`).

## Results (smoke scale — regenerate with `make demo`)

*18 Hamiltonians (TFIM / Heisenberg / molecular-like, n ∈ {3,4}), t = 2.0, r = 3 — the
laptop-scale smoke configuration.* The commutator ordering attains the highest fidelity at
the **lowest** number of anticommuting adjacencies (0.33 vs 2.33 for random), a **1.99×
reduction in infidelity vs random** at the same gate proxy. Reported-scale numbers (n up to
8, 8 instances/family, tight CIs) come from `make full-run` with
[`configs/full.yaml`](code/configs/full.yaml). The fidelity-vs-gate-budget frontier and the
per-family breakdown are written to `results/summary.json` and plotted in `figures/`.

### Library — `code/src/topoham/`, file by file

| File | Contribution |
|---|---|
| [`pauli.py`](code/src/topoham/pauli.py) | Pauli strings on `n` qubits, the O(n) `commute()`/`anticommute()` parity test, and the exact single-Pauli rotation `e^{-iθP} = cos θ·ψ − i sin θ·(Pψ)`. **The kernel the whole simulator rests on** — and the math the browser demo re-implements. |
| [`hamiltonians.py`](code/src/topoham/hamiltonians.py) | The three benchmark families — transverse-field Ising, Heisenberg (XX+YY+ZZ), and `molecular_like` random local 2-body sums — emitted as `[coeff, paulistring]` term lists. |
| [`commutator_graph.py`](code/src/topoham/commutator_graph.py) | The commutator graph: nodes are terms, edges are anticommuting pairs. Exposes anticommutation density, greedy commuting-group colouring, and the set-invariant features the learned router reads. |
| [`env.py`](code/src/topoham/env.py) | First-order Trotter statevector evolution plus the exact `scipy.linalg.expm` and Krylov `expm_multiply` references — the audited ground truth every fidelity is scored against. |
| [`policies.py`](code/src/topoham/policies.py) | All five term orderings behind one interface — random, coefficient, locality, **commutator (the contribution)**, and the learned scikit-learn router. All compared at a matched gate budget. |
| [`metrics.py`](code/src/topoham/metrics.py) | Fidelity, observable (Z₀) error, gate proxy, anticommuting-adjacency counts, and bootstrapped 95% CIs. |

## How to use

```bash
cd code
make install         # numpy / scipy / networkx / scikit-learn / matplotlib (CPU-only)
make test            # 24 tests incl. Pauli-rotation ↔ expm cross-check to 1e-9
make demo            # ~3 s: writes results/summary.json
make tables figures  # results/main_results.{tex,md}, figures/*.pdf
make audit           # readiness gate: traceable numbers, no forbidden claims
make full-run        # reported-scale configuration
```

Or open [`index.html`](index.html) directly in any browser to explore the method interactively — it
re-implements the exact single-Pauli rotations and the statevector Trotter sweep in pure JavaScript, so
what you see on the page is the same math as the Python package.

## GPU / acceleration note

The reference implementation is **CPU-only and runs everywhere**. The architecture exposes
optional accelerated backends ([`code/requirements-optional-gpu.txt`](code/requirements-optional-gpu.txt)):
a **cuQuantum** statevector / tensor-network Trotter path for `n > ~20` qubits (where the
dense `expm` reference no longer fits), a **PyTorch-Geometric** message-passing GNN over the
commutator graph as a drop-in for the ordering policy, and a differentiable Trotter-error
objective for end-to-end training. None are required to reproduce the results.
