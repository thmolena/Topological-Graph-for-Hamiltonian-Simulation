## Summary

Simulating the time evolution `e^{-iHt}` of a many-body Hamiltonian is a founding
application of quantum computers, and the first-order Lie–Trotter product formula remains
its dominant primitive. For a Hamiltonian in Pauli form `H = Σ_j c_j P_j`, the per-step cost
of a first-order schedule is exactly one rotation per term, so the gate count `L·r` is
invariant under the term ordering `π`—and under whether that ordering varies between steps.
A long line of empirical work has found reordering to be a modest lever; this work explains
why, and identifies the lever that is not modest.

We introduce the **commutator error form**, the exact leading-order Trotter error operator
`E_π = Σ_{a<b} [h_{π(a)}, h_{π(b)}]`, assembled matrix-free as a signed sum of Pauli strings
over the commutator graph and validated against dense matrix exponentials to `1e-9`.
Reordering only **re-signs** the commutators in `E_π`; its Hilbert–Schmidt norm is *exactly*
ordering-invariant whenever the edge commutators are distinct, so **no fixed ordering escapes
the `O(t^2/r)` first-order rate** — a precise, provable form of the "ordering is within noise"
folklore. The freedom that does matter is the per-step **schedule**: a step-alternating
**antithetic** schedule cancels `E_π` exactly and converges at second order, `O(t^3/r^2)`, at
the **identical** `L·r` rotation count. The effect is measured against exact statevector
references on 120 structured Hamiltonian instances, every quantity reported as a mean with a
95% confidence interval.

Every number in this README is generated by the included code and transcribed verbatim from
[`submission/code/results/summary.json`](submission/code/results/summary.json); each value
traces to that artifact rather than to manual entry.

## Principal contributions

1. **The commutator error form.** The exact leading-order error operator `E_π` is assembled
   in `O(L^2 n)` as a signed sum of Pauli strings over the commutator graph `G(H)`, validated
   against the dense pair-commutator sum to `1e-9`.
2. **Ordering impotence (theorem).** Reordering only re-signs `E_π`; its Hilbert–Schmidt norm
   is exactly ordering-invariant when the edge commutators are distinct, and within a factor
   `1.085` of its floor otherwise across the 120 instances. No fixed ordering escapes the
   first-order rate.
3. **A free second-order schedule.** The antithetic schedule cancels `E_π` and converges at
   second order at the identical gate budget; combined with a commuting-clique compression of
   `G(H)` it reaches a target fidelity at `1.62×` fewer rotations and reaches `0.999` where
   first-order Trotter does not.
4. **A learned schedule router.** An interpretable logistic-regression router over ten
   commutator-graph features selects the gate-optimal schedule per instance with `85%`
   leave-one-instance-out accuracy, recovering near-oracle gate efficiency.

## Method

```
 H = Σ c_j P_j ──► commutator graph ──► schedule σ ──────────► product formula ──► fidelity
   (Pauli terms)   (edges = anticommute,  (order · fold · clique,    (Π e^{-i c_j P_j t/r})    vs e^{-iHt}
                    cliques + colours)     matched L·r rotations)                                (expm / Krylov)
                          │                        │
                          ▼                        ▼
                 leading error form        ordering re-signs E_π (impotence);
                 E_π = Σ_{a<b}[h,h]         antithetic fold cancels E_π (free 2nd order)
```

1. **Pauli algebra** ([`pauli.py`](submission/code/src/topoham/pauli.py)) — Pauli strings on `n`
   qubits with an `O(n)` commutation parity test, the exact single-Pauli rotation
   `cos θ·ψ − i sin θ·(Pψ)`, and the `pauli_product` kernel `PQ = ω R`. A test asserts the
   rotation matches `scipy.linalg.expm(−iθP)` to `1e-9`.
2. **Commutator graph and error form**
   ([`commutator_graph.py`](submission/code/src/topoham/commutator_graph.py),
   [`error_form.py`](submission/code/src/topoham/error_form.py)) — the graph, its commuting
   cliques, the matrix-free leading-error operator `E_π`, the collision/impotence statistics,
   and the ten listing-order-invariant features the router reads.
3. **Schedules** ([`schedules.py`](submission/code/src/topoham/schedules.py)) — first-order,
   antithetic (the free second-order fold), and symmetric (Strang) folds over a coefficient or
   commuting-clique ordering, with exact rotation accounting.
4. **Audited reward** — fidelity of the schedule's state against the exact `e^{-iHt}` reference
   (`scipy.linalg.expm`, cross-checked against Krylov `expm_multiply`).

## Main results

Reported scale: 120 structured instances (transverse-field Ising, Heisenberg, and random
molecular-like families; `n ∈ {4,…,8}` qubits; eight seeded draws each), evolution time
`t = 1.0`. Transcribed verbatim from
[`submission/code/results/summary.json`](submission/code/results/summary.json).

**Matched-cost snapshot at the reference budget (`r = 6`, 76 rotations).** The first-order
schedules share the identical 76-rotation budget; the antithetic fold matches it and still
reaches higher fidelity. `rate p` is the fitted convergence exponent.

| schedule | order | fidelity | ±95% CI | rotations | rate p |
|---|---|---|---|---|---|
| random (1st order) | 1st | 0.9149 | 0.0131 | 76 | 1.99 |
| coefficient (1st order) | 1st | 0.9636 | 0.0075 | 76 | 1.97 |
| clique (1st order) | 1st | 0.9650 | 0.0073 | 76 | 2.03 |
| **antithetic (2nd, free)** | 2nd | **0.9806** | 0.0062 | 76 | **4.07** |
| symmetric (2nd, Strang) | 2nd | 0.9989 | 0.0003 | 141 | 4.03 |

At the identical 76-rotation budget the antithetic schedule reaches mean fidelity `0.9806`
against `0.9636` for the best first-order ordering — a reduction of mean infidelity from
`0.0364` to `0.0194` (a factor `1.9`), for free. The rate column is the same effect as a
convergence exponent: `≈2` for every first-order schedule, `≈4` for both folds.

**Rotations to a target fidelity (matched-cost efficiency).** "—" marks a target not reached
within the swept step grid.

| schedule | F ≥ 0.9 | F ≥ 0.99 | F ≥ 0.999 |
|---|---|---|---|
| random (1st order) | 76 | 304 | — |
| coefficient (1st order) | 51 | 152 | — |
| clique (1st order) | 51 | 152 | — |
| **antithetic (2nd, free)** | 76 | **101** | **203** |
| symmetric (2nd, Strang) | 94 | 94 | 188 |

To reach fidelity `0.99` the best first-order ordering needs `152` rotations and the folds
`94–101` — a `1.62×` reduction. To reach `0.999`, **no first-order schedule succeeds within
the swept budget**, while the antithetic and symmetric schedules reach it at `203` and `188`
rotations. Because the convergence rates differ, the matched-cost gap widens with budget.

**Ordering impotence.** Sampling orderings per instance and forming the exact leading-error
spectral norm, a median ordering sits within a factor `1.085` of the per-instance minimum and
the coefficient ordering within `1.067`. On the transverse-field Ising and Heisenberg families
*every* anticommuting edge maps to a distinct Pauli string, so the leading error is exactly
ordering-invariant; only the dense molecular-like family has any ordering-reducible part (96%
invariant). The learned router attains `85%` leave-one-instance-out accuracy and recovers
near-oracle gate efficiency by folding only the instances that need it.

The convergence (infidelity vs `r`), the matched-cost frontier (infidelity vs rotation count),
the per-family decomposition, and the impotence/collision panels are written to
[`submission/code/results/summary.json`](submission/code/results/summary.json) and rendered in
[`submission/code/figures/`](submission/code/figures).

## Reproduction

A single command regenerates every figure, table, macro, and main number from one seeded run.
Invoke it from `submission/code`:

```bash
cd submission/code
topoham-reproduce --config configs/full.yaml
```

The run is fully determined by seeded draws (`seed = 0`), so the reported numbers are recovered
exactly; only the provenance fields (timestamp, runtime, peak memory) vary between runs. The
equivalent Makefile path from a clean checkout is:

```bash
cd submission/code
make test            # Pauli kernel (1e-9), error-form (1e-9), antithetic 2nd-order, clique exactness
make full-run        # reported scale: 120 instances → results/summary.json
make tables figures  # results/macros.tex, tab_*.tex, figures/*.pdf
make audit           # readiness gate: traceable numbers, integrity flags, vetted claims
```

`make demo` substitutes the smoke configuration for a sub-minute check.

## Scope

The second-order advantage is a small-step-size effect with an honest crossover: at coarse
steps the antithetic fold can be worse than a good first-order ordering. Systems are small
(`n ≤ 8`) because an exact statevector reference is formed; the convergence-rate argument is,
however, size-independent. Families are structured proxies rather than production-mapped
molecular Hamiltonians, and cost is the realised rotation count, abstracting rotation synthesis
and device connectivity. All results are classical CPU statevector simulations, reported
without hardware validation.
